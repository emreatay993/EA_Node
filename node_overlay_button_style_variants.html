Node Overlay Toolbar — Button Style Variants

The floating toolbar that appears above (or below) a node on hover already works well in terms of placement. These variants explore how the buttons themselves look and feel — shape, density, labelling, and hover treatment — while keeping the same four actions (Rename, Duplicate, Open, Delete) and the same viewport behavior. Each stage renders on a representative dark canvas so you can compare them apples-to-apples.

Variant 1 is the current shipped look (from commit 277f864) — it's the reference point. Pick the variant you'd like to move toward, and I'll convert it into an implementation plan that maps onto GraphNodeFloatingToolbar.qml and GraphSurfaceButton.qml.

Ownership Which node does the toolbar belong to?

On dense graphs the toolbar can float in the gap between two nodes, leaving the user guessing which one will be renamed or deleted. The cures below are orthogonal to the style variants — you can layer any cue onto any of the seven styles below.

How to choose

  • Want less visual weight? Variants 2, 5, or 6.
  • Want it to feel more "premium"? Variant 3 (glass) or 7 (CTA + rail).
  • Want it to feel like a real toolbar? Variant 4 (segmented).
  • Stay conservative, just polish? Tweaks on top of Variant 1 are cheapest.
