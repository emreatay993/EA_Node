## Locked Defaults

- `ansys-dpf-core` remains optional. Startup without DPF must preserve all non-DPF workflows.
- The DPF plugin version must always match the exact installed `ansys.dpf.core.__version__` string.
- Plugin version state is app-global and must not rewrite saved `.sfe` projects.
- The first wave is hybrid staged: full operator coverage plus curated workflow helpers, not a literal mirror of all public `ansys.dpf.core` modules.
- The library is workflow-first:
  - operators grouped by DPF family
  - helpers grouped by workflow role
  - raw or noisy surfaces deferred under `Advanced > Raw API Mirror`
- Generated descriptors must retain original DPF source path, resolved family path, and stability metadata.
- P03 must establish disjoint operator/helper catalog seams before P04 and P05 run in parallel.
- When a later packet changes a seam asserted by an earlier test, the later packet owns the inherited regression anchor inside its own write scope.
