# ANSYS_DPF_FULL_PLUGIN_ROLLOUT P02: Contract Expansion

## Objective

- Expand the node and graph contracts so autogenerated DPF operators and helper callables can publish multi-type pins, callable-backed source metadata, and normalized DPF type identities without hand-written one-offs.

- `P01` is marked `PASS` in [ANSYS_DPF_FULL_PLUGIN_ROLLOUT_STATUS.md](./ANSYS_DPF_FULL_PLUGIN_ROLLOUT_STATUS.md).

## Required Behavior

- Add a first-class contract for multi-type input compatibility so autogenerated DPF pins are not forced into one `data_type`.
- Add callable binding metadata for helper constructors, factories, and mutators alongside the existing operator metadata contract.
- Normalize DPF type identifiers for high-value object families and define the generic fallback object-handle path for everything else.
- Update registry validation and graph compatibility logic to honor the new contract without regressing existing non-DPF nodes.

## Non-Goals

- No taxonomy rollout yet; that belongs to `P03`.
- No operator or helper descriptor generation yet; that belongs to `P04` and `P05`.
- No runtime or serializer adoption yet; that belongs to `P06`.
