Port Value Locking — UX Variants

Concept: Primitive inputs (int, float, bool, string) are always shown as connectable ports.
Double-click port dot or entering a value locks the port (no longer eligible for connections).
Ctrl + Double-click & MMB + LMB toggle locked-port visibility  |  MMB + RMB toggle optional port visibility.

Try it: Double-click any port dot to lock/unlock  ·  Type a value in a field  ·  Use the buttons below each node to simulate gestures
A · Lock & Slide
Locking a port replaces the dot with a padlock icon. The port row stays full height, but a value field appears in place of a wire slot. Toggling hides locked rows with an accordion collapse animation.
Port intent is explicit ("this is pinned") Lock icon is universally understood Locked ports still consume row height Icon may feel heavy at small scale

B · Compact Chip Recommended
Locking collapses the port row to a slimmer "chip" size — dot shrinks to a tiny pip, label fades, and the value field compresses. Unlocked rows keep full height. Toggle hides chips entirely for maximum graph clarity.
Visual weight reflects semantic weight Locked / free ports are instantly scannable No clutter — chips are unobtrusive Chips are harder to click-to-unlock

C · Ghost + Slash
Locked ports keep their row height but the dot becomes translucent with a diagonal slash overlay (matches the existing "inactive" port visual language already in the editor). Label gets a strikethrough. Toggle hides ghosted rows.
Consistent with existing inactive-port style Zero new visual vocabulary to learn Subtle — locked state can be missed Strikethrough may feel harsh

Double-click port dot
Toggle port between connectable and value-pinned. Pinned port shows inline editor and cannot receive a wire.
Type a value into field
Entering any non-default value auto-locks the port after a short debounce, keeping the graph uncluttered without an extra click.
Ctrl+DblClick & MMB+LMB
Toggle visibility of all locked/pinned ports. Use to de-clutter the graph when many primitive values are set.
MMB+RMB
Toggle visibility of optional ports (e.g. DPF operator optional inputs). Separate from locked-port toggle — independent control.
