## Required Behavior

- Introduce an explicit unresolved DPF node state that keeps saved node title, layout, properties, port labels, exposed ports, and connectivity available from persistence data.
- Keep missing-plugin placeholders visible and serializable while blocking execution and live schema editing.
- Surface an unavailable reason tied to the missing DPF plugin contract instead of silently dropping the node.
- Rebind placeholder nodes to live plugin-backed behavior when the DPF plugin becomes available again.
- Update inherited serializer and graph-projection regression anchors in place when the unresolved-node contract changes.

## Non-Goals

- No editable offline schema snapshot mode.
- No recovery path for arbitrary missing non-DPF plugins beyond what the DPF placeholder contract naturally generalizes.
- No autogenerated operator rollout yet.
