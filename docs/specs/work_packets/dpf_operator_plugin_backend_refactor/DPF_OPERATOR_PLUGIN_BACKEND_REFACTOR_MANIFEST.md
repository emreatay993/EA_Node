## Locked Defaults

- This packet set is backend preparation only. Do not ship broad autogenerated DPF operator exposure in this wave.
- `ansys-dpf-core` remains optional. Startup without DPF must preserve all non-DPF workflows.
- The foundation is operators-first. Non-operator `ansys.dpf.core` reflection remains out of scope.
- Existing handwritten DPF nodes must continue to work while the backend contract is generalized underneath them.
- Generated-port semantics must carry enough source metadata to distinguish required inputs, optional inputs, omitted operator defaults, literal values, and mutually exclusive input groups.
- Missing-plugin DPF nodes remain visible as read-only placeholders with saved values and connectivity, but they cannot execute or reveal live plugin-backed property editing until the plugin is available again.
- Preserve `.sfe` compatibility, existing non-DPF node IDs, and current plugin package boundaries.
- Later packets run strictly sequentially; no non-bootstrap packet shares a wave.
- When a later packet changes an earlier packet's asserted seam, the later packet inherits and updates that earlier regression anchor inside its own write scope.
