## Ownership Rules

- Pick exactly one primary subsystem packet for every future UI packet.
- If a change crosses subsystem boundaries, list the secondary packet docs explicitly and inherit their required tests instead of broadening one packet into an omnibus rewrite.
- Update the owning subsystem doc when a public entry point, invariant, or forbidden shortcut changes.
- Keep packet-owned QML on focused bridges and helper seams rather than reintroducing raw host globals.
