5 alternatives — each shows 3 nodes: idle (pending), running (active), completed (done)

Mirrors existing failure pulse pattern

Track edge states by mapping node_started/node_completed events to upstream/downstream edges. Requires a new active_edge_lookup in the bridge.

Recommendation: Combine A + E (Hybrid)

Use Alternative A (Glowing Border Pulse) as the primary running indicator — it reuses the existing failure pulse architecture in GraphNodeChromeBackground.qml with minimal new code, and is visible at all zoom levels.

Optionally layer in Alternative E's elapsed timer below nodes for profiling insight. The timer is a small additive feature that provides real diagnostic value without cluttering the visual.

Shared backend for all alternatives: Every option requires the same Python-side change — a running_node_lookup and completed_node_lookup dictionary on GraphCanvasStateBridge, populated by reacting to node_started / node_completed events in RunController.handle_execution_event(). The protocol already emits these events — no worker changes needed.
